# Contributing to f_solar

Thank you for considering contributing to the f_solar project! To ensure a smooth workflow, please follow the guidelines outlined below.

## Git Workflow Guidelines

### Branch Naming Standards
- Use feature branches for new features: `feature/your-feature-name`
- Use bugfix branches for bug fixes: `bugfix/your-bugfix-name`
- Use hotfix branches for urgent fixes in production: `hotfix/your-hotfix-name`

### Commit Message Format
- Use the following format for commit messages:  
  `TYPE: Short description of the change`
- Where TYPE can be:  
  - `feat`: A new feature  
  - `fix`: A bug fix  
  - `docs`: Documentation only changes  
  - `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)  
  - `refactor`: A code change that neither fixes a bug nor adds a feature  
  - `test`: Adding missing or correcting existing tests  
  - `chore`: Changes to the build process or auxiliary tools and libraries such as documentation generation

### Pull Request Checklist
- [ ] I have followed the branch naming standards  
- [ ] My commits follow the commit message format  
- [ ] I have added tests to cover my changes  
- [ ] I have updated the documentation (if necessary)  
- [ ] The code is formatted correctly

### Merge Strategy
- Use **Squash and Merge** for merging feature branches to main.  
- Ensure that all checks are successful before merging.  

Thank you for contributing!
