# AI Code Guidelines

## Team Rules
- **Respectful Communication:** Team members are expected to communicate respectfully and constructively at all times.
- **Collaboration:** Work collaboratively to solve problems and share knowledge.
- **Ownership and Accountability:** Take responsibility for your contributions and ensure follow-through on assigned tasks.

## Branch Naming Standards
- **Feature Branches:** Use the format `feature/description` (e.g., `feature/add-login`).
- **Bug Fix Branches:** Use the format `bugfix/description` (e.g., `bugfix/fix-typo`).
- **Hotfix Branches:** Use the format `hotfix/description` when urgent fixes are needed (e.g., `hotfix/urgent-fix`).

## Code Review Requirements
- **Pull Requests:** All code must be submitted via a pull request for review before being merged into the main branch.
- **Review Process:** At least one teammate must review and approve the pull request.
- **Feedback:** Provide constructive feedback, focusing on code quality, security, and adherence to team guidelines.

## Best Practices for Handling AI-Generated Code
- **Review Thoroughly:** Always review AI-generated code as you would with human-written code; check for logical consistency and potential errors.
- **Understand the Code:** Ensure you understand the AI-generated code before integrating it into the project. Don’t just insert it blindly.
- **Document Changes:** Provide clear comments or documentation on AI-generated code to explain its purpose and any modifications made.
- **Integrate Gradually:** Test AI-generated code in isolation before full integration to ensure it works as expected and doesn’t introduce new bugs.

## Conclusion
Following these guidelines will help ensure high-quality code and foster a collaborative working environment while integrating AI-generated contributions effectively.
